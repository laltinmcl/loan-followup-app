# System Architecture

## Overview

The Loan Follow-Up Management System follows a modern **three-tier architecture** with a React PWA frontend, Node.js REST API backend, and PostgreSQL database. The system is designed for **offline-first field operations** and **real-time synchronization**.

## Data Flow

### Online Mode
```
User Action вҶ’ React Component вҶ’ API Service вҶ’ HTTP Request вҶ’ Express Route
  вҶ’ Controller вҶ’ Service (Business Logic) вҶ’ Prisma вҶ’ PostgreSQL
  вҶ’ JSON Response вҶ’ React State Update вҶ’ UI Re-render
```

### Offline Mode (Field Visits)
```
User Action вҶ’ React Component вҶ’ Local State вҶ’ IndexedDB (Cache)
  вҶ’ Service Worker (Background Sync Queued)
  вҶ’ [When Online] вҶ’ Sync API вҶ’ Express вҶ’ Prisma вҶ’ PostgreSQL
  вҶ’ Sync Response вҶ’ Update IndexedDB вҶ’ UI Notification
```

### Real-time Updates
```
WebSocket Connection вҶҗвҶ’ Express WebSocket Server вҶҗвҶ’ PostgreSQL (via LISTEN/NOTIFY)
  вҶ’ Push to connected clients вҶ’ React State вҶ’ UI Update
```

## Key Design Decisions

### 1. Offline-First Architecture
- Service Worker caches static assets (App Shell pattern)
- IndexedDB stores loan data and visit records locally
- Background Sync API queues mutations when offline
- Conflict resolution: "last-write-wins" with server timestamp
- Photos stored as Base64 in IndexedDB when offline, uploaded when online

### 2. Stage Workflow Engine
- Finite state machine implemented in the backend service layer
- Each stage has: entry actions, exit actions, auto-reminder triggers
- Transitions logged to activity_log table for audit trail
- Configurable via constants (no code changes needed for stage adjustments)

### 3. PWA Strategy
| Asset | Cache Strategy |
|-------|---------------|
| App Shell (HTML/CSS/JS) | Cache-first (pre-cached at install) |
| API Responses | Network-first, fallback to cache |
| Visit Photos | Network-only (upload), cached after |
| Static Images | Cache-first |
| Loan Data | IndexedDB (manually synced) |

### 4. Security
- JWT-based authentication (single-role)
- All API routes protected except auth endpoints
- Rate limiting on all endpoints
- Input validation via Zod schemas
- CORS configured for frontend domain only
- Photo uploads validated for type and size

## Scalability

- **Horizontal scaling**: Stateless API servers behind a load balancer
- **Database**: Read replicas for reporting queries
- **File storage**: CDN for photo delivery
- **Caching**: Redis for session store and frequent queries (future)
- **Background jobs**: Queue-based processing for imports/exports (future)
